1.1 Creación de la Base de Datos

Modelo Entidad–Relación (conceptual)

Se modelaron entidades como Productos, Proveedores, Clientes y Movimientos de stock (Entradas/Salidas), definiendo claves y relaciones para garantizar integridad.

Implementación en SQL Server

Se creó la base de datos y tablas físicas a partir del modelo, contemplando tipos de datos, PK/FK e índices.

-- Ejemplo de creación (referencial)

CREATE TABLE Productos (
  IdProducto INT IDENTITY PRIMARY KEY,
  Nombre NVARCHAR(100) NOT NULL,
  Precio DECIMAL(10,2) NOT NULL,
  Stock INT NOT NULL DEFAULT 0

2. Conclusiones y Sugerencias

Retos enfrentados

  • Modelo de datos: definición de relaciones (Productos, Proveedores, Entradas/Salidas) y normalización.
  • Integridad: claves, restricciones y consistencia en operaciones de stock.

Nuevas tecnologías utilizadas

  • SQL Server (DDL/DML, SSMS).
  • Git (flujo de commits, ramas, merges).
  • GitHub (colaboración, issues, Pages).

Sugerencias

  • Ampliar pruebas automatizadas y validaciones.
  • Implementar control de inventario con triggers o procedimientos almacenados.
  • CI/CD simple para despliegues de la app.
